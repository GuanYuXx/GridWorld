## 5. 為什麼 Noise 最多只能到 0.5？而且照理來說拉高 noise 可以影響每個狀態到下個狀態的方向才對，為何單次迭代感覺方向沒有被 noise 干擾？
這是一個非常好的問題，也是理解這個基礎 GridWorld 強化學習（Markov Decision Process, MDP）模型核心設計的關鍵！

### 為什麼 Noise 最多只能到 0.5？
在目前這個強化學習環境（MDP 機制）中，`noise` 代表的是**「滑走」到非預期方向的機率**。這裡的數學模型定義為：
當你選擇一個方向（如「向上」），你實際的移動結果是：
*   **照原意方向走**的機率 = `1.0 - 2 * noise`
*   **滑向左側**的機率 = `noise`
*   **滑向右側**的機率 = `noise`

因為所有的機率加起來必須是 1，而且任何一個事件的機率不能小於 0。
如果你的 `noise` 超過 0.5，那麼 `1.0 - 2 * noise` 就會變成負數（例如 noise=0.6，照原意的機率就是 -0.2），這在數學上不合理。
當 `noise = 0.5` 時，代表你選擇「向上」，你「向上」的機率是 0%，一定會以 50% / 50% 隨機向左或向右滑。因此 0.5 是這個數學模型的極限。

### 為什麼調高 Noise，單次迭代箭頭方向感覺沒被干擾？
這點要區分**「演算法的評估」**跟**「實際玩遊戲的過程」**。

*   **網格上顯示的箭頭，代表的是「策略 (Policy)」**：策略指的是「在考慮了所有意外（Noise 的可能性）之後，系統認為在這個格子**『打算走哪裡』**最好」。
*   **價值迭代計算的是「期望值」**：Value Iteration 並不是真的把一個代理人 (Agent) 丟進去跑，然後看他跑到哪裡。它是在算數學公式：計算走到目標的期望分數有多少。

如果畫面上只有「終點」而沒有「陷阱」，就算你把 `noise` 調到 0.4（代表你只有 20% 機率照原意走），系統還是會認為：「既然四面八方都很安全，我還是『瞄準』終點走就是最好的選擇」。因此，儘管計算出的 Value 數值會因為容易滑開而「變得很低」，但**最佳的應對策略（箭頭方向）依然是筆直瞄準目標**，所以箭頭看起來沒變。

**💡 如何看出 Noise 對「策略箭頭」的影響？（你可以試試看！）**
要看到高 Noise 導致策略改變，您需要設置**「陷阱 (Trap)」**。
1.  在畫面中央放一個高價值的「終點 (Goal)」，旁邊緊貼著放幾個負價值的「陷阱 (Trap)」。
2.  **低 Noise (例如 0.0)** 時：旁邊的格子會大膽地直接指向終點，因為它確信自己 100% 不會滑進陷阱。
3.  **高 Noise (例如 0.4)** 時：旁邊的格子的箭頭可能會**刻意避開終點的方向，甚至選擇撞牆**。因為系統計算後發現：「如果我硬要瞄準終點，因為我很容易打滑，我有 40% 的機率會慘跌進陷阱裡！相較之下，我寧願去撞安全的牆壁（原地踏步扣時間分），也比掉進陷阱好。」這時候，您就會看到 Noise 明顯地「干擾」並改變了決策的方向！

## 6. 那把step reward調高會有甚麼影響
這是一個非常關鍵的參數！在強化學習（特別是這個 GridWorld 的例子）中，`Step Reward`（每步的獎勵/懲罰）決定了代理人（Agent）**「對時間的急迫感」**。

在這個模型中，每走一步，Agent 就會獲得這個 `Step Reward`。目前的預設值是 **-1.0**，也就是說，每走一步，Agent 就會被「扣 1 分」。

### 6.1. 為什麼預設是 -1.0（負數）？
這是在鼓勵 Agent **「越快走到終點越好」**。
*   目標（Goal）的分數是 +10。
*   如果 Agent 繞遠路，走了 8 步才到，他的總得分是：`10 + 8 * (-1) = 2`。
*   如果 Agent 走最短路徑，只花 3 步就到了，他的總得分是：`10 + 3 * (-1) = 7`。
因為強化學習的目的是「追求最高的分數」，所以 Agent 會自然而然地選擇**最短、最直接的路線**，以減少這 -1 分的無謂扣分。

### 6.2. 如果把 Step Reward 調高（例如調到 0 或正數）會發生什麼事？

**A. 調到 0.0（不扣分）：沒有時間壓力**
如果設定為 0，代表「逛街」不用付任何代價。
*   此時 Agent **完全沒有動機去走最短路徑**。
*   只要它確信最終一定會走到終點（拿到 +10），它繞 100 步跟走 3 步對它來說「價值是一模一樣的」。
*   這時算出來的策略（箭頭）可能會變得非常隨意，甚至在原地繞圈圈，因為在它眼中條條大路通羅馬，它不趕時間。

**B. 調成正數（例如 +1.0）：鼓勵永無止境地逛街**
這會出現一個非常有趣（且通常不是我們想要的）的反效果。
*   每走一步就賺 1 分，那這遊戲的最高境界就不再是「跑到 +10 的終點」了。
*   相反地，Agent 會想盡辦法**「避開終點」**！因為一旦走到終點，遊戲就結束了（Reward 停止發放）。
*   它會在網格上永無止境地兜圈子，因為只要活著一步就能拿 1 分，走到終點拿 10 分反而會讓它無法繼續「刷分」。
*   此時你會看到箭頭全部指向牆壁或在空地繞圈，盡可能地遠離終點。

### 總結
`Step Reward` 就像是給 Agent 的**「生活費」**。
*   **負數生活費（-1）**：每天都在噴錢，必須盡快跑到終點拿獎金。
*   **正數生活費（+1）**：每天醒來都有錢拿，傻子才去終點結束這種好日子，大家一起在網格裡散步到永遠！

## 9. 障礙物數量最多只能放n-2個，另外我要確定現在每個策略都要有上下左右4種策略，即是遇到牆角或是障礙物也是嗎？
關於「障礙物數量最多只能放 n-2 個」的限制已經實作完成！

而關於「每個狀態都要有 4 種策略（包含撞牆）」，這其實是強化學習 GridWorld 轉移模型 (Transition Model) 處理「撞牆」的標準做法，原本的系統**已經有實作這個邏輯**了：
1. **策略只顯示在可行走的格子上**：終點和障礙物本身「不具備」策略，因為掉進終點就結束了，而障礙物不可進入，所以它們上面不會顯示方向箭頭。
2. **撞牆 (Bump) 的處理**：如果代理人 (Agent) 目前的格子**緊貼著障礙物或邊界**，它**確實可以選擇**把方向指向那個障礙物。數學模型中，如果 Agent 選擇撞向障礙物或牆壁，轉移結果是**「退回原格 (原地踏步)」**，並照樣被扣除 1 次 Step Reward 的分數。
這也是為什麼加入 Noise 之後策略會轉彎的原因：如果正前方有終點、但左邊有陷阱，Agent 有時候會故意選擇「右轉去撞牆」。因為它寧願撞牆原地踏步一次（扣分解），也不願意冒險直走、如果不小心滑跤掉進左邊的陷阱。
